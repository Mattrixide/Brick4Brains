Executive Summary
Brick for Brains v0.1 — Planning Phase

Project Overview

Planning

The Brick for Brains is an autonomous combat robotics system that uses computer vision to track robots in an 8x8ft beetleweight arena, make real-time strategic decisions, and control our robot to push the opponent into a corner pit — all without human intervention during a match. The system combines an overhead OAK-D Pro camera, ArUco marker tracking, depth-based enemy detection, a behavior-tree strategy engine, and WiFi-based ESP32 motor control to achieve fully autonomous combat.

Key Technical Decisions

Source: PRD Section 7
Area Decision Rationale
Camera OAK-D Pro, 1080p/60fps, fixed-focus Standard 69 HFOV covers 8ft arena from 7ft height. Fixed-focus avoids hunting. 1080p/60fps is the best balance of resolution and frame rate.
Mounting 6-7ft above arena floor Covers full arena with standard lens FOV. Provides ~240 px/ft resolution at 1080p — sufficient for 3-inch ArUco markers.
Tracking ArUco 4x4_50 (3-inch markers) Fewest bits = most robust at distance. Only ~7 unique IDs needed. Markers appear as ~60px at 1080p from 7ft.
Enemy Detection Depth thresholding + background subtraction Hybrid approach: depth finds objects above floor plane, appearance-based fallback for low-profile robots.
Communication WiFi UDP to ESP32, <20ms target 5-15ms RTT meets target. No bridge ESP32 needed (unlike ESP-NOW). Simple asyncio integration.
Stack Python (DepthAI + FastAPI) + React Native OAK-D support, OpenCV ArUco built-in, WebSocket support for live data, rich interactive UI.
Exposure Manual, 400-500 microseconds Freezes robot motion at 8 ft/s to <1 pixel of blur. Arena LED lighting compensates for reduced exposure.
Sensor Fusion Complementary filter (MVP), EKF (Phase 2) Complementary filter is simpler to implement. EKF upgrade in Phase 2 for better accuracy.
