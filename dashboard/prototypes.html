CV Tracking Prototype prototypes/cv-tracking/
Complete

Webcam-based computer vision tracking validating ArUco detection, color tracking, background subtraction, Kalman filtering, and simulated chaser bot.

What it validates
  • ArUco 4x4_50 detection speed and reliability at various distances
  • Threaded camera capture model (379% FPS improvement over blocking reads)
  • MOG2 background subtraction for markerless enemy detection
  • Kalman filter value for smoothing and prediction during occlusion
  • Combined pipeline FPS budget (ArUco + BGSub + Kalman in one loop)
  • Resolution vs FPS tradeoffs across 480p, 720p, and 1080p
How to run
cd prototypes/cv-tracking pip install -r requirements.txt python main.py
Keyboard controls
1 ArUco mode 2 Color mode 3 BGSub mode 4 Combined mode t Toggle trail k Toggle Kalman f Cycle resolution e / d Exposure +/- q Quit
Dependencies
opencv-contrib-python numpy
TX15 Drive Test prototypes/drive-test/
In Progress

TX15 SBUS motor control with ArUco ID#1 tracking (50mm marker) via external webcam. Currently blocked on TX15 USB driver issue on Windows 11.

What it validates
  • SBUS protocol generation and transmission over USB serial to TX15
  • ArUco-guided autonomous movement (spin test using marker heading)
  • Real-world distance measurement from known marker size (50mm)
  • End-to-end pipeline: camera → detection → motor command
How to run
# Dry-run mode (no TX15 hardware required) cd prototypes/drive-test pip install -r requirements.txt python main.py # Live mode with TX15 connected python main.py --port COM3
Keyboard controls
SPACE Auto spin test W Forward A Left S Backward D Right X Stop + / - Speed q Quit
Dependencies
opencv-contrib-python numpy pyserial
Current blocker

TX15 USB Serial Driver (Windows 11) — STM32 VID:0483 PID:5740 CDC composite device fails to enumerate with usbser driver (error code 10). The TX15 is not recognized as a COM port. Investigating alternative drivers and firmware-level workarounds.

Troubleshooting checklist (try when TX15 is available):

  1. Check EdgeTX settings: System → Hardware → USB → change to “USB-VCP (CLI)”
  2. Try a USB 2.0 port (not USB 3.0)
  3. Install STSW-STM32102 VCP driver from ST.com
  4. Try ImpulseRC Driver Fixer tool
  5. Use Zadig to assign “USB Serial (CDC)” driver
  6. Update/downgrade EdgeTX firmware
  7. Try a custom INF file (requires disabling driver signature enforcement)
  8. Use a USB-to-UART adapter connected to JR module bay trainer port
  9. Boot into Linux live USB to verify TX15 works at OS level
  10. Try Windows safe mode with no third-party USB drivers
  11. Check for Windows 11 USB driver conflicts in Event Viewer
  12. Contact RadioMaster support with device manager screenshots
